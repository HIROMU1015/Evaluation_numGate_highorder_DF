# DF Hamiltonian + PF: GR から DF への置換と比較軸

この文書は、従来の「グルーピングしたハミルトニアン（GR）」に基づく評価を、**double factorization（DF）したハミルトニアン**へ置き換えたときに、
どの誤差予算の下で、どの量をどう見積もり、どの指標で GR と DF を比較しているかを、**理論と処理のつながり**として説明する。

---

## 要点（このプロジェクトで伝えたいこと）

1. 以前は **GR（グルーピング）**を対象にしていたが、本プロジェクトでは **DF したハミルトニアン**を対象にした。
2. PF + QPE の目標エネルギー誤差は **化学的精度（CA）の 1/10**とし、DF の近似（rank 打ち切り）に伴う CCSD(T) エネルギー誤差は **CA の 1/100**になるように選ぶ。
3. PF のトロッター誤差は、**真のハミルトニアンの基底エネルギー**ではなく、**DF ハミルトニアン `H_DF` の基底エネルギー `E0_DF` を基準**にした誤差として、摂動論に基づく評価で求める（DF 上の simulation error）。参考として真の基底エネルギーも小サイズでは求められるが、誤差の「定義上の基準」ではない。
4. DF のレイヤー指標は、DF のブロック構造 `U D U^\dagger` に沿って回路を捉え、**非Clifford な `RZ/RZZ`（および同等の Z 系位相回転）**だけを抜き出して、**support 彩色（衝突しない集合の層分け）**で並列レイヤー数を数える。U 部分と D 部分は分けて集計できる。
5. 最終的な計算コストの比較では、H10 近辺で DF が GR を下回る可能性がある。経験的に H8 前後で「PF 1 ステップ当たりのレイヤー数（彩色指標）」が DF 側に有利へ転じる兆候がある。

---

## 1. 比較軸（同じ“量”を数える）

GR と DF は回路の構成が本質的に違うため、比較指標を揃えないと差の解釈ができない。
本プロジェクトのレイヤー比較は次の抽象度に揃える。

- GR: 「Z 文字列（あるいはそれに同等な位相回転）」を、support（作用する論理量子ビット集合）で層分けした **論理レイヤー数**。
- DF: `H_DF` 由来の回路に含まれる **非Clifford な `RZ/RZZ`** を抽出し、同じく support の衝突だけで層分けした **論理レイヤー数**。

この「support 彩色レイヤー」は、分解器・最適化・ルーティングの違いを一旦外して、
**“位相回転をどれだけ並列に置けるか”** という構造だけを比較するための指標である。

---

## 2. 全体の処理フロー（何を入力し何を出すか）

入力は主に (i) 系サイズ（H-chain の長さ）と (ii) 積公式（PF の次数ラベル）である。
このプロジェクトで比較したいのは、同じ誤差予算の下での **GR と DF の「レイヤー指標」のスケーリング**である。

1. H-chain を定義し、1 体・2 体積分からハミルトニアンを定める。
2. 2 体項を DF 形式で近似し、DF ハミルトニアン `H_DF` を構成する。
3. DF の rank（ブロック数 `L`）を、CCSD(T) 誤差が `CA/100` 以下となるように選ぶ。
4. `H_DF` を対象に PF を適用し、`E0_DF` 基準の PF 誤差を摂動論推定で評価する。
5. DF の回路表現 `U D U^\dagger` から、非Clifford `RZ/RZZ` の support 彩色レイヤー数（U と D を分割可）を数える。
6. 同じ抽象度の彩色レイヤー指標で、GR と DF を比較する。

このうち「**ハミルトニアンを DF して積公式（PF）に渡す**」処理は、概ね次の流れで実装されている（関数名は例）。

1. 分子積分（`one_body_integrals`, `two_body_integrals`, `constant`）を入力として DF 分解する  
`trotterlib.df_trotter.decompose.df_decompose_from_integrals(...)`  
内部では OpenFermion の `low_rank_two_body_decomposition(...)` と `spinorb_from_spatial(...)` を用いて、`(lambda_ell, G_ell)` の列を得る。
2. 各 `G_ell` を対角化して `U_ell` と `D_ell`（占有数基底の対角係数）に分け、`U D U^\dagger` ブロック列へ変換する  
`trotterlib.df_trotter.ops.build_df_blocks(model)`（対角化は `diag_hermitian(...)`）  
`U_ell` の回路分解では Qiskit Nature の `BogoliubovTransform`（環境によっては `fermionic_gaussian_decomposition_jw`）を用いる。
3. （必要なら）1体項も同様にガウス変換 + 対角回転のブロックへ落とす  
`trotterlib.df_trotter.ops.build_one_body_gaussian_block(one_body)`
4. ブロック列（`blocks`）に対して PF の係数列 `w` と対称ステップ列を生成し、積公式回路を構築する  
`trotterlib.df_trotter.circuit.build_df_trotter_circuit(blocks, time, num_qubits, pf_label, ...)`  
この中で `trotterlib.product_formula._get_w_list(pf_label)` と `trotterlib.pf_decomposition.iter_pf_steps(num_terms, w_list)` を呼び、`(term_idx, weight)` の列を走査して `U D U^\dagger` ブロックを順に適用する。

## 3. DF ハミルトニアンと `U D U^\dagger` ブロック

### 3.1 DF で何が起きるか（2体項の低ランク化）

電子の 2 体相互作用は（スピン軌道基底で）非常に多くの項を持つが、DF（低ランク近似）では
2 体の寄与を「1 体演算子の二乗の和」として表現する。

概念的には

```
H = H1 + H2
H2  ~  sum_{ell=1..L}  lambda_ell * (A_ell)^2
```

のように書き換える（`L` が DF の rank、`A_ell` が 1 体の Hermitian 演算子）。

ここで `L` を小さくする（打ち切る）ほど、DF 近似誤差は大きくなるが、回路の複雑さは下がる。

### 3.2 `U D U^\dagger` になる理由（対角化と占有数）

各 `A_ell` は 1 体演算子なので、ある基底変換 `U_ell` により「占有数演算子 `n_p` の線形結合」に対角化できる。
その結果、`(A_ell)^2` の時間発展は

```
exp(-i tau * lambda_ell * (A_ell)^2)
  = U_ell * exp(-i tau * lambda_ell * (sum_p eta_{ell,p} n_p)^2) * U_ell^\dagger
  = U_ell * D_ell(tau) * U_ell^\dagger
```

となり、`U D U^\dagger` ブロックとして回路化できる。

`D_ell(tau)` は占有数（0/1）だけで決まる対角位相なので、
JW 写像の下で **Z 回転（1 量子ビット）**と **ZZ 回転（2 量子ビット）**の組み合わせに落ちる。
一方 `U_ell` は fermionic Gaussian 変換であり、回路としては主に（論理的に）2 量子ビットのガウス型ゲート列に分解される。

この構造が「DF は `U` と `D` を分けてコスト評価できる」理由になっている。

## 4. 誤差予算と rank 選択（CA/10 と CA/100 の関係）

### 4.1 最終目標（PF + QPE の許容誤差）

最終的に欲しいエネルギー推定精度を

- `epsilon_E = CA / 10`

と置く（CA の 1/10）。

この `epsilon_E` を（PF + QPE を含む）最終的なエネルギー精度の誤差予算として扱う。

### 4.2 DF 近似誤差（CCSD(T) を基準に CA/100）

DF は元ハミルトニアンからの近似なので、**DF 近似そのものの誤差**が大きいと、
後段の PF 誤差評価や資源見積もりが、モデル誤差に支配されて解釈しにくくなる。

そこで DF の打ち切り（rank）を、化学計算側の精度指標として

- `| E_CCSD(T)(DF) - E_CCSD(T)(exact) | <= CA / 100`

を満たすように選ぶ。

直観的には、最終目標 `CA/10` に対して DF モデル誤差をさらに 1 桁小さく抑えることで、
PF の誤差解析を「DF ハミルトニアン上で」進めても、モデル誤差に汚染されにくくする狙いがある。

### 4.3 実際の選び方（threshold scan -> 最小 rank）

実務上は、DF の閾値（または rank）をいくつか試し、

1. その DF 近似で CCSD(T) エネルギーを評価する
2. exact（フル）との差の絶対値を取る
3. 目標（CA/100）を満たす中で、できるだけ小さい rank を採用する

という選択になる。

## 5. PF 誤差の定義（DF 基底エネルギー基準）

ここで重要なのは「誤差の基準」である。

- 本プロジェクトで扱う PF のトロッター誤差は、**`H_DF` の基底エネルギー `E0_DF` を基準**に定義している。
- したがって、ここでの誤差は「真のハミルトニアン `H_exact` に対する誤差」ではなく、**DF 近似後のモデル `H_DF` 上の simulation error**である。
- 誤差量の評価自体は、摂動論に基づく推定（perturbation estimator）を用いる。

真の `H_exact` の基底エネルギーは、比較や検算のために小サイズでは求められるが、
ここでのトロッター誤差の「基準」としては用いていない（主張 3 のポイント）。

## 6. DF のレイヤー数（非Clifford `RZ/RZZ` の support 彩色）

### 6.1 なぜ「non-Clifford」に絞るのか

フォールトトレラント量子計算では、Clifford 操作は比較的安価であり、
支配的になるのは T ゲート等に対応する **非Clifford 回転**であることが多い。

そこでレイヤー指標も

- 「非Clifford な位相回転」に対応する部分

に絞って評価する。

### 6.2 2つの“深さ”の概念（compiled 深さ vs 彩色深さ）

同じ「RZ 深さ」と言っても、少なくとも 2 種類の測り方がある。

1. **compiled 深さ（DAG 依存）**  
   回路をある基底ゲート集合へ分解・最適化した後に、DAG 上で深さを数える。
   これは分解器や最適化の選択に依存し、GR 側の指標とは抽象度が違う。
   DAG（directed acyclic graph）は、ゲートの依存関係を表す有向非巡回グラフで、深さは依存グラフ上の最長鎖（クリティカルパス）として定義される。

2. **support 彩色深さ（論理的スケジューリング）**  
   非Clifford な `RZ/RZZ` を「support（作用する量子ビット集合）」として抽出し、
   support が重ならないものを同じレイヤーに詰める（グラフ彩色／層分け）。
   これは GR 側のスケジューリング指標と同じ抽象度で比較できる。

本プロジェクトで GR と DF の比較をする際は (2) を採用する。

### 6.3 DF での `RZ/RZZ` レイヤーの内訳（U と D の寄与）

DF の 1 ブロック `U D U^\dagger` は、レイヤー指標としては「U 部分」と「D 部分」に分けられる。

- **D 部分**  
  `D` は占有数演算子の多項式なので、回路としては `RZ`（1 量子ビット）と `RZZ`（2 量子ビット）に展開できる。
  非Clifford 角の `RZ/RZZ` を抽出し、その support（{q} または {q0,q1}）を彩色して層数を数える。

- **U 部分**  
  `U` は fermionic Gaussian 変換で、論理的には 2 量子ビット相互作用の列に分解される。
  その中で非Clifford な位相回転に相当する部分（実装上は Z 回転や連続パラメータを持つ 2 量子ビット操作）を
  `RZ/RZZ` に相当する Z 系位相回転として扱い、同様に support 彩色で層数を数える。

これにより、DF の「非Clifford Z 彩色レイヤー」の合計は

```
DF_total_nonclifford_z_coloring_depth
  = (U 部分の彩色レイヤー) + (D 部分の彩色レイヤー)
```

として評価できる。

### 6.4 GR 側の指標と揃える意味

GR 側は、グルーピング後の可換ブロックを「Z 文字列の位相回転（論理 `RZ`）」へ落とし、support 彩色で層数を数える。
このとき basis-change（U）を無視しているのは、比較指標が「Z 回転の並列化可能性」だからである。

DF 側も同じ抽象度に揃えるなら、DF でも「非Clifford `RZ/RZZ` の support 彩色」を比較軸にする必要がある。
そうしないと、差が「分解手法（GR vs DF）」ではなく「コンパイルや分解器の違い」に汚染される。

## 7. DF が H10 付近で有利になり得る理由（解釈）

この節は厳密な証明ではなく、観測される傾向を説明するための直観である。

- DF の D 部分 `D_ell(tau)` は `RZ` と `RZZ` の集合に落ちる。
  `RZZ` は 2 量子ビット support を持つため、support 彩色深さは「2 量子ビット相互作用の並列化可能性」に支配され、
  1 ブロック当たりの層数は概ね `O(N)` に抑えられる余地がある（N は量子ビット数）。
- CCSD(T) 目標（CA/100）で打ち切った結果として、DF の rank `L` が **`O(n_orb)` 程度**まで落ちる領域があり得る。
  このとき `U` の（論理）層数が `O(n_orb)`、`D` の彩色深さも概ね `O(n_orb)` と見なせるので、
  1 ステップ全体の `U D U^\dagger` コストは
  `O(L * n_orb) = O(n_orb^2)` スケールとして振る舞い得る、というのがここで言いたい点である。

経験的に、ある程度のサイズ（例: H8 近辺）で DF の 1 ステップ当たりレイヤー指標が GR と同程度になり、
さらに大きいサイズで DF が有利へ転じる可能性がある、という主張はこの直観と整合する。

## 8. 限界と注意点

- **彩色指標は楽観的**  
  support 彩色は「理想的に並列化できる」という下界寄りの見積もりであり、配線制約・CNOT ラダー・ルーティングなどは含まない。
- **non-Clifford 判定は t に依存しうる**  
  角度が偶然 `pi/2` の整数倍に近いと non-Clifford 判定が変わる。ただし一般の `t` では多くの回転が非Clifford と判定されやすい。
- **真の基底エネルギーは“基準”ではない**  
  ここでのトロッター誤差は `E0_DF` 基準であり、真の `E0_exact` 基準の誤差ではない。真の基底エネルギーは検算用途にはなるが、固有値問題が重くなるためサイズには限界がある。

---

## まとめ

- GR から DF へ対象を置き換え、(i) DF の rank を CCSD(T) 誤差（CA/100）で選び、(ii) PF 誤差を `E0_DF` 基準で評価し、(iii) レイヤー数は非Clifford `RZ/RZZ` の support 彩色で定義して比較した。
- GR vs DF を「レイヤー数」で比較するなら、両者を support 彩色という同じ抽象度に揃えるのが本質である。
